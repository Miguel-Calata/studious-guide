- Instalar
    
    ### 1. Python (El motor principal)
    
    - **Dónde descargarlo:** Ve a [python.org/downloads](https://www.python.org/downloads/) y descarga la última versión para Windows.
    - **⚠️ EL PASO CRÍTICO:** Al abrir el instalador, fíjate en la parte inferior de la primera ventana. Verás una casilla que dice **"Add python.exe to PATH"** (Agregar Python al PATH). **DEBES marcar esa casilla** antes de darle a "Install Now". Si no la marcas, Windows PowerShell no sabrá qué hacer cuando escribas `python` o `pip`.
    - **Cómo verificar que quedó bien:** Abre PowerShell y escribe `python --version`. Si te responde con la versión instalada (ej. `Python 3.12.x`), estás listo.
    
    ### 2. Visual Studio Code (Tu centro de mando)
    
    Te servirá para editar los `.txt`, ver cómo van quedando los `.md` y ejecutar la terminal, todo en una sola pantalla.
    
    - **Dónde descargarlo:** Ve a [code.visualstudio.com](https://code.visualstudio.com/) y descarga el instalador.
    - **Instalación:** Dale a "Siguiente" a todo.
    - **Extensiones recomendadas:** Una vez abierto, ve al panel de extensiones (los cuadritos en la barra lateral izquierda) y busca **Python** (de Microsoft) e instálala. Esto te ayudará a leer los scripts fácilmente.
    
    ### 3. Las Librerías de Python (Los "complementos")
    
    Tu script `extraer_pdf_crudo.py` requiere una librería específica para leer y transformar los PDFs a Markdown.
    
    - Abre Visual Studio Code.
    - Abre la terminal integrada presionando `Ctrl + Shift + ñ` (o ve arriba al menú `Terminal > New Terminal`). Asegúrate de que diga **PowerShell**.
    - Escribe el siguiente comando y presiona Enter:
    
    ```
    pip install pymupdf4llm
    ```
    
    *Deja que termine de descargar e instalar los paquetes. Esto toma solo un par de minutos.*
    
    Una vez que tengas estas tres cosas, tu laptop Windows estará 100% preparada. Podrás abrir la carpeta `Compendios SAM` en Visual Studio Code, abrir la terminal ahí mismo y empezar a correr todos los comandos de las Fases 0 a la 6 tal como los estructuramos.

    ## El flujo completo, paso a paso
    
    ### FASE 0 — Preparación (una sola vez por patología, antes de empezar)
    
    ```
    1. Deposita todos los PDFs de la patología en pdfs_pendientes/
    2. Clasifica cada PDF en una de estas tres categorías:
    ```
    
    | PDF | Tipo | Señal clave | Prompt |
    | --- | --- | --- | --- |
    | BMJ Best Practice | Guía de punto de atención | Secciones colapsables, sin comité de autores | `v3_CORREGIDO` |
    | KDIGO, WHO, ESC, AHA, Renal Association extensa | Guía completa | Tiene comité, metodología GRADE, votación | `v5_guias_completas` |
    | Nature, Lancet, NEJM, JAMA | Artículo de revista | Tiene Abstract, afiliaciones, volumen/número | `ARTICULOS_UNIFICADO` |
    
    ### FASE 1 — Extracción (Gemini, una fuente a la vez)

    
    ```
    Por cada PDF de la patología:
    
    1. Chat NUEVO en Gemini 2.5 Pro en https://aistudio.google.com/prompts/new_chat
       - Grounding: APAGADO
       - Temperature: 0.1 (guías) o 0.2 (artículos)
       - Output: 65536
    
    2. Adjunta el PDF
    
    3. Pega el prompt de extracción correspondiente
       (v3_CORREGIDO, v5, o ARTICULOS_UNIFICADO según la tabla de arriba)
    
    4. Si Gemini dice [CONTINÚA — Pendiente desde: "X"]:
       → Escribe "Continúa" en el MISMO chat
       → El protocolo anti-ciclo ya está activo, no necesitas hacer nada más
    
    5. En el MISMO chat, pega PROMPT_AUDITORIA.txt
       → Gemini compara su extracción contra el PDF y añade lo que faltó
    
    6. Descarga cada respuesta como .md
       → Guarda en raw_parts/<patologia>/01.md, 02.md, 03.md...
    ```

    ## 2. El Flujo Completo: Paso a Paso
    
    ### FASE 0 — Preparación (Una sola vez por patología)
    
    1. Deposita todos los PDFs de la patología en la carpeta `pdfs_pendientes\`.
    2. Clasifica cada PDF en una de estas tres categorías:
    
    | **PDF / Tipo** | **Señal clave** | **Prompt a usar** |
    | --- | --- | --- |
    | **BMJ Best Practice** (Guía de punto de atención) | Secciones colapsables, sin comité de autores | v3_CORREGIDO |
    | **KDIGO, WHO, ESC, AHA** (Guía completa) | Tiene comité, metodología GRADE, votación | v5_guias_completas |
    | **Nature, Lancet, NEJM, JAMA** (Artículo de revista) | Tiene Abstract, afiliaciones, volumen/número | ARTICULOS_UNIFICADO |
    
    ### FASE 1 — Extracción en Gemini (Una fuente a la vez)
    
    1. Abre un Chat NUEVO en Gemini 2.5 Pro (Google AI Studio). Configura: Grounding APAGADO, Temperature 0.1 (guías) o 0.2 (artículos), y Output 65536.
    2. Adjunta el PDF.
    3. Pega el prompt de extracción correspondiente según la tabla anterior.
    4. Si Gemini dice `[CONTINÚA — Pendiente desde: "X"]`, escribe "Continúa" en el MISMO chat. El protocolo anti-ciclo ya está activo.
    5. En el MISMO chat, pega `PROMPT_AUDITORIA.txt`. Gemini comparará su extracción contra el PDF y añadirá lo que faltó.
    6. Descarga cada respuesta como archivo `.md` y guárdalos en `raw_parts\<patologia>\01.md`, `02.md`, etc.
    
    ### FASE 2 — Verificación y Limpieza
    
    Confirma que no quedaron duplicados en cada fuente ejecutando este comando (cambia la ruta según tu archivo):
    
    ```
    (Select-String -Path "raw_parts\AKI\AKI_BMJ_01.md" -Pattern "^#.*").Matches.Value | Group-Object | Where-Object Count -gt 1 | Select-Object Name
    ```
    
    Si no sale nada, está limpio. Si salen títulos repetidos, verifica manualmente si es un error de ciclo o un subtítulo válido. Si es un ciclo, debes rehacer esa fuente.
    
    ### FASE 3 — Unión
    
    Ejecuta el script de Python para unir los archivos:
    
    ```
    python unir_extraccion.py
    ```
    
    Eliges la patología y te generará el archivo final (ej. `markdowns\AKI.md`).
    
    Luego, limpia los artefactos de interfaz (si aplica) con este comando:
    

- `PROMPT_EXTRACCION_v3_CORREGIDO.txt`
    
    ```jsx
    Eres un transcriptor clínico de precisión absoluta. Tu única tarea es
    trasladar el contenido del PDF adjunto a Markdown sin perder nada y sin
    reorganizarlo en una plantilla externa. No interpretas, no resumes, no
    reclasificas. Copias la estructura de pensamiento del documento, no la tuya.
    
    ════════════════════════════════════════════════════════════════════════
    🚨 PROTOCOLO ANTI-CICLO (LEE ESTO PRIMERO — CAUSA DE FALLO CONOCIDA)
    ════════════════════════════════════════════════════════════════════════
    
    Un fallo confirmado en extracciones anteriores: al pedir "continúa", el
    modelo a veces vuelve a desarrollar encabezados que ya había completado
    en su respuesta anterior dentro de la MISMA conversación, en vez de avanzar
    estrictamente desde el punto donde se quedó. Esto duplicó secciones enteras
    de un documento (se llegó a repetir la estructura completa, capítulo 1 al
    11, dos veces).
    
    REGLA OBLIGATORIA EN CADA CONTINUACIÓN:
    Antes de escribir una sola palabra, revisa mentalmente — encabezado por
    encabezado — qué ya desarrollaste en tus respuestas anteriores DENTRO DE
    ESTA MISMA CONVERSACIÓN. Localiza el punto exacto donde terminaste (el
    marcador "[CONTINÚA — Pendiente desde: ...]" te lo indica). Comienza
    ÚNICAMENTE desde ese punto. Si tienes cualquier duda sobre si ya
    desarrollaste un encabezado, NO LO VUELVAS A ESCRIBIR — omitirlo por
    precaución es preferible a duplicarlo, porque un vacío puntual se corrige
    después con una auditoría dirigida, mientras que un duplicado infla el
    archivo sin aportar nada y degrada todo el proceso posterior.
    
    ════════════════════════════════════════════════════════════════════════
    PRINCIPIO RECTOR: LA ESTRUCTURA ES DEL DOCUMENTO, NO TUYA
    ════════════════════════════════════════════════════════════════════════
    
    NO uses una plantilla predefinida de secciones. En su lugar:
    
    1. Recorre el PDF de principio a fin, en el orden exacto en que aparece.
    2. Cada encabezado, subtítulo o bloque temático que el PDF ya tiene se
       convierte en un encabezado Markdown (##, ###, ####, #####) con el
       mismo nombre o su traducción directa al español.
    2b. JERARQUÍA DE SUBSECCIONES (específico para UpToDate): este
    documento tiene hasta 4-5 niveles de subsección anidada. Preserva
    cada nivel con su indentación correspondiente en Markdown (##, ###,
    ####, #####). Si un bloque de texto pertenece a una subsección
    específica, nunca lo "sueltes" al nivel del encabezado padre —
    queda bajo su subtítulo exacto, con su nivel correcto.
    3. Si el documento tiene bloques especiales ("Practical tip", "Evidence:",
       "Debate:", casos clínicos, algoritmos con etiquetas condicionales),
       esos bloques se convierten en subtítulos propios, preservando la
       etiqueta original entre paréntesis si aplica, y todo su contenido se
       transcribe completo.
    4. Nunca fusiones dos bloques que el documento presenta por separado.
    5. Nunca decidas que algo "no es importante" para omitirlo. Si dudas,
       inclúyelo.
    
    ════════════════════════════════════════════════════════════════════════
    REGLAS DE FIDELIDAD
    ════════════════════════════════════════════════════════════════════════
    
    DOSIS Y FÁRMACOS: dosis exacta con unidades, vía, tiempo de administración,
    frecuencia, duración de efecto, monitoreo, contraindicaciones, cita numérica.
    Si el mismo fármaco aparece en escenarios distintos con dosis distintas, se
    transcribe en cada escenario por separado.
    
    TABLAS: se reproducen completas, mismo número de filas y columnas.
    
    ALGORITMOS Y ÁRBOLES DE DECISIÓN: cada rama es su propio bloque con su
    condición de entrada explícita, sus intervenciones en orden, y su criterio
    de escalada. Nunca colapses ramas distintas en una lista genérica.
    
    CAJAS ESPECIALES: "Practical tip", "Evidence:", "Debate:", notas al pie,
    advertencias y casos clínicos se transcriben íntegros bajo su propio
    subtítulo, en el lugar exacto donde aparecen — no se mueven ni se resumen.
    
    VALORES NUMÉRICOS: todo umbral, punto de corte, porcentaje, intervalo de
    tiempo o valor de laboratorio, con sus unidades originales.
    
    CITAS: cada cita numérica del PDF ([1], [75], etc.) se conserva junto al
    dato que respalda.
    
    FIGURAS, VIDEOS Y ELEMENTOS VISUALES: tabla al final con identificador,
    tipo, descripción, ubicación, y sección donde aparece.
    
    ════════════════════════════════════════════════════════════════════════
    ENCABEZADO OBLIGATORIO
    ════════════════════════════════════════════════════════════════════════
    
    # TRANSCRIPCIÓN FIEL: [nombre exacto de la patología/tema según el PDF]
    
    **Fuente:** [título, organización o autor si el PDF lo indica]
    **Tipo de documento:** [guía clínica / capítulo / artículo / monografía]
    **Guías o criterios citados:** [lista las que identifiques]
    
    ---
    
    [A partir de aquí, sigue la estructura nativa del documento]
    
    ════════════════════════════════════════════════════════════════════════
    CONTROL DE EXTENSIÓN
    ════════════════════════════════════════════════════════════════════════
    
    Si el contenido es muy extenso, detente al límite natural de una sección
    (nunca a la mitad de una tabla o un algoritmo) y escribe exactamente:
    [CONTINÚA — Pendiente desde: "nombre del siguiente encabezado del PDF"]
    ═══════════════════════════════════════════════════════════════════════════
    ```

- LAS ÚLTIMAS RECOEMDACIONES QUE ME DIO GEMINI - PARA CONSIDERAR Y EVALUAR
    
    Como arquitecto de este sistema, me toca hacer el rol de "abogado del diablo". Aunque la arquitectura de SAM v9 es una obra maestra de *Prompt Engineering*, el flujo de trabajo operativo tiene **4 desventajas críticas** que van a chocar de frente contra la realidad cuando empieces a procesar patologías masivas.
    
    Aquí están las fisuras del modelo y por qué podrían causarte problemas en tu día a día:
    
    ### 1. El Muro de Tokens (Cuotas de Uso en la Interfaz Web)
    
    Esta es la amenaza más inmediata para tu flujo.
    
    - **El problema:** Tu mega-archivo Markdown tiene +150,000 tokens. Si creas un chat nuevo para la sección 01, subes el archivo; para la sección 03 en Claude, creas otro chat y lo vuelves a subir; para la 06, otro chat.
    - **La realidad:** Las interfaces web (tanto Claude Pro como Gemini Advanced) tienen límites estrictos de contexto. Claude, por ejemplo, reduce drásticamente el número de mensajes permitidos si le subes documentos gigantes. Si abres 4 chats nuevos en Claude y le inyectas 150k tokens a cada uno en menos de una hora, vas a agotar tu cuota diaria (el famoso *"Please wait until X:XX PM"*) antes de terminar una sola patología.
    
    ### 2. El "Eco Ciego" (Falta de Retroalimentación Real)
    
    El `MAPA_ECOS` es una solución brillante, pero opera bajo una presunción peligrosa: asume que la IA hizo un trabajo perfecto en el chat anterior.
    
    - **El problema:** Es una "memoria estática programada", no una "memoria real". Si al generar la sección 02 (Clasificación), la IA por error omitió un detalle pequeño sobre la clasificación neonatal, la sección 03 (Fisiopatología) asumirá ciegamente que el dato ya existe (porque el `MAPA_ECOS` así se lo dicta) y simplemente pondrá `→ Ver Clasificación`.
    - **El resultado:** Ese dato clave se perderá en el limbo y no aparecerá en el compendio final.
    
    ### 3. Fricción Operativa (Sobrecarga de Copy-Paste)
    
    El objetivo de SAM era reducir tus 5 horas de edición, pero este modelo traslada parte de esa carga al área operativa.
    
    - **El problema:** Para una sola patología tienes que ejecutar el script, abrir 11 archivos de texto, alternar entre dos plataformas distintas (Gemini y Claude), crear unos 9-10 chats nuevos, adjuntar el documento pesado 9-10 veces, y copiar/pegar los resultados de vuelta a tu editor (Visual Studio Code o Notion).
    - **El resultado:** Sigue siendo un proceso altamente manual. Si te distraes y pegas el prompt 08 en el chat equivocado, rompes la secuencia.
    
    ### 4. Inconsistencia Micro-Semántica (El síndrome de las múltiples personalidades)
    
    Al estar en chats aislados, cada modelo arranca "en frío".
    
    - **El problema:** Aunque sigan las mismas reglas de formato (Leyes de SAM v9), la IA no puede ver cómo redactó el párrafo anterior. En un chat podría usar consistentemente el término "LRA" (Lesión Renal Aguda), y en el siguiente chat, al procesar una guía británica, podría empezar a usar "FRA" (Falla Renal Aguda) o "AKI".
    - **El resultado:** Cuando unas todo en tu Mega-Markdown final, el tono y la nomenclatura pueden sentirse ligeramente "parchados" en las transiciones entre secciones.
    
    ### ¿Cómo mitigamos esto en el futuro cercano?
    
    El modelo que creaste es el **techo máximo** de lo que se puede lograr copiando y pegando en las interfaces web (ChatGPT/Claude/Gemini web).
    
    Para solucionar estas 4 desventajas de raíz, el siguiente paso evolutivo natural (un posible "SAM v10") sería conectar tu script de Python directamente a las **APIs de Google (Gemini) y Anthropic (Claude)**. De esa forma:
    
    1. El script le pasaría dinámicamente lo que *realmente* generó la sección 2 a la sección 3.
    2. Todo sucedería en la terminal de tu computadora sin que tú tengas que hacer un solo copy-paste.
    3. Pagarías por token consumido (centavos), saltándote los bloqueos de cuota de la interfaz web.
    
    Por ahora, para la v9, mi recomendación táctica es: **Ten cuidado con tus cuotas de uso en Claude**. Si ves que te bloquean rápido por subir tantas veces el archivo de 150k tokens, tendrás que agrupar más secciones en un mismo chat y depender de que mi capacidad de atención aguante la instrucción de no repetir cosas.

    Al conectar tu script de Python directamente a los "cerebros" de Google y Anthropic, dejas de pagar las suscripciones mensuales de las versiones web ($20 USD c/u) y pasas a un modelo de "pago por uso" (*Pay-as-you-go*). Pagarás exactamente por los tokens que consumas.

    ### 1. Tarifas Base de las APIs (Costo por 1 Millón de tokens)

    A diferencia de la versión web, en la API debemos enviar el documento de 150,000 tokens en cada una de las 11 peticiones para que la IA tenga el contexto.

    - **Total de Escritura (Output):** ~800 tokens de texto generado × 11 secciones = ~8,800 tokens.
